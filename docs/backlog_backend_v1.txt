ATLASIP — BACKEND BACKLOG v1.0
===========================================================
Document de référence à placer dans /doc/backlog_backend_v1.txt
Ce fichier regroupe tous les bugs, specs, correctifs et tâches liés au BACKEND d’AtlasIP.
Le parser v2 est exclu (géré dans un document séparé).

Structure : Criticité (P0 → P3) → Catégorie → Bug détaillé.


===========================================================
P0 — CRITICITÉ CATASTROPHIQUE (BLOQUANT)
===========================================================

P0-RDAP-001
Catégorie : RDAP
Description : Le backend RDAP renvoie des résultats incomplets ou vides pour certains RIR.
Reproduction : Interroger des IPs APNIC / AFRINIC.
Observé : Champs manquants (org, abuse, contacts).
Attendu : Normalisation complète et cohérente.
Impact : Résultats OSINT incorrects.
Sprint : Backend v1.

P0-RDAP-002
Catégorie : RDAP
Description : Timeout RDAP non géré correctement.
Reproduction : Interroger une IP lente.
Observé : L’UI freeze ou renvoie un résultat vide.
Attendu : Timeout propre + fallback WHOIS.
Impact : Analyse bloquée.
Sprint : Backend v1.

P0-WHOIS-001
Catégorie : WHOIS
Description : Le fallback WHOIS ne se déclenche pas en cas d’échec RDAP.
Reproduction : IP sans endpoint RDAP.
Observé : Erreur silencieuse.
Attendu : WHOIS automatique.
Impact : Résultats manquants.
Sprint : Backend v1.

P0-WHOIS-002
Catégorie : WHOIS
Description : Parsing WHOIS instable selon le format du registrar.
Reproduction : IPs variées (RIPE, ARIN, LACNIC).
Observé : Champs incohérents.
Attendu : Parsing robuste multi‑formats.
Impact : Données OSINT fausses.
Sprint : Backend v1.

P0-DNS-001
Catégorie : DNS
Description : Résolution DNS non implémentée.
Reproduction : Entrer "google.fr".
Observé : Aucun enregistrement A/AAAA.
Attendu : Résolution DNS complète.
Impact : Fonctionnalité manquante.
Sprint : Backend v2.

P0-NETWORK-001
Catégorie : Réseau
Description : Pas de gestion centralisée des erreurs réseau.
Reproduction : Couper la connexion.
Observé : Panique ou erreur non catchée.
Attendu : Gestion d’erreur propre.
Impact : Crash potentiel.
Sprint : Backend v1.
