ATLASIP — BACKLOG EXPORTS v1.0
===========================================================
Document de référence à placer dans /doc/backlog_exports_v1.txt
Ce fichier regroupe tous les bugs, specs, correctifs et tâches liés aux EXPORTS d’AtlasIP.
Le parser v2 est exclu (géré dans un document séparé).

Structure : Criticité (P0 → P3) → Catégorie → Bug détaillé.


===========================================================
P0 — CRITICITÉ CATASTROPHIQUE (BLOQUANT)
===========================================================

P0-EXPORT-001
Catégorie : Export JSON
Description : Export JSON renvoie un objet vide ou incomplet.
Reproduction : Analyser une IP → Export JSON.
Observé : JSON vide ou partiel.
Attendu : JSON complet, conforme aux résultats affichés.
Impact : Perte de données OSINT.
Sprint : Exports v1.

P0-EXPORT-002
Catégorie : Export CSV
Description : Export CSV casse les colonnes si un champ contient une virgule.
Reproduction : Résultat RDAP avec "Org, Inc".
Observé : CSV décalé.
Attendu : Échappement correct des champs.
Impact : CSV inutilisable.
Sprint : Exports v1.

P0-EXPORT-003
Catégorie : Export Maps
Description : Export Maps ne fonctionne pas si plusieurs IPs sont sélectionnées.
Reproduction : Sélectionner 2 IPs → Export Maps.
Observé : Erreur silencieuse.
Attendu : Ouverture de la carte avec plusieurs points.
Impact : Fonctionnalité cassée.
Sprint : Exports v1.

P0-EXPORT-004
Catégorie : Export Copy
Description : Copy renvoie un texte tronqué.
Reproduction : Résultats longs → Copy.
Observé : Seulement une partie copiée.
Attendu : Copie complète.
Impact : Perte d’information.
Sprint : Exports v1.

SPÉCIFICATIONS EXPORTS (HORS PARSER)
===========================================================

1. Export Copy
- copier toutes les colonnes visibles
- respecter l’ordre du DataGrid
- conserver les sauts de ligne
- copier l’intégralité des résultats

2. Export CSV
- respecter l’ordre du DataGrid
- échapper les virgules
- échapper les guillemets
- ajouter BOM UTF‑8
- ignorer les colonnes masquées

3. Export TSV
- tabulations correctes
- même logique que CSV

4. Export JSON
- pretty-print
- suppression des champs null
- ordre stable des champs
- cohérence avec le backend

5. Export Maps
- support IPv4
- support IPv6
- support multi‑IP
- support ports (optionnel)
- ouverture dans Google Maps ou OpenStreetMap

6. Export Details
- format textuel lisible
- sections claires (RDAP / WHOIS / DNS)
- titres et séparateurs
