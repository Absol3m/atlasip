1. Introduction
Ce document définit le backlog complet pour la réécriture du parser d’AtlasIP.
Il sert de référence unique pour :

les développeurs

les tests unitaires

la QA

les prompts Claude Code

la roadmap des sprints

Le parser actuel présente des incohérences majeures.
Ce document définit la spec complète, les bugs, les règles, les cas limites, et les tâches à réaliser.

2. Objectifs du Parser v2
Le parser doit :

être strict

être cohérent

être prévisible

être testable

être OSINT‑friendly

gérer les cas mixtes

bloquer les erreurs

fournir un feedback clair

fonctionner avec du texte libre

extraire IPs, hostnames, URLs, headers

3. Types d’entrées supportées
3.1 IPv4
Format strict

4 octets

Chaque octet ∈ [0, 255]

3.2 IPv6
Format standard

Utiliser std::net::Ipv6Addr::from_str

3.3 Hostname
Regex stricte

Pas de double point

Pas de caractères interdits

3.4 URL
Parser via url::Url

Extraire hostname

Extraire port (optionnel)

Extraire protocole (optionnel)

3.5 Texte libre
Extraction d’IP

Extraction de hostname

Extraction d’URL

Extraction de headers email

Ignorer les mots normaux

5. Règles de validation globale
5.1 Une seule entrée invalide → tout bloque
bouton Analyze = gris

badge = rouge

message d’erreur global

aucune analyse possible

5.2 Toutes les entrées valides → OK
bouton Analyze = bleu

badge = mode correct

compteur cohérent

6. Détection du mode
Cas	Mode
IPv4 only	IPv4 DETECTED
IPv6 only	IPv6 DETECTED
Hostname only	HOSTNAME DETECTED
Mix IP + hostname	MIXED MODE
Mix IP + URL	MIXED MODE
Mix hostname + URL	MIXED MODE
Mix IPv4 + IPv6	MIXED MODE
Erreur	INVALID
