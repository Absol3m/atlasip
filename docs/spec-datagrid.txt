# AtlasIP — DataGrid Specification (v1.0)
Author: Absol3m
Language: English
Scope: Rules, behavior, and requirements for the main results table.

--------------------------------------------------
1. Overview
--------------------------------------------------
The DataGrid is a core component of AtlasIP’s frontend. It displays the results
of IP lookups and must support large datasets (500+ rows) without performance
issues.

The DataGrid must be:
- Fast (virtualized rendering)
- Editable (IP column)
- Configurable (column visibility, order, width)
- Persistent (user preferences saved)
- Export-friendly
- Keyboard-friendly

The DataGrid is NOT a simple table; it is an interactive workspace.

--------------------------------------------------
2. Data Model
--------------------------------------------------
Each row corresponds to an IpRecord returned by the backend.

Columns must match the backend’s COLUMNS definition:
- ip
- hostname
- resolved_name
- country
- state
- city
- owner_name
- org_name
- netname
- cidr
- range_start
- range_end
- abuse_emails
- contact_emails
- phone
- fax
- address
- whois_source
- created_at
- updated_at

Rules:
- Empty values displayed as “—”
- Email lists displayed as “email1; email2”
- Long text truncated with tooltip

--------------------------------------------------
3. Column Behavior
--------------------------------------------------

### 3.1. Resizing
- Columns are resizable by dragging edges.
- Minimum width: 80px
- Maximum width: 600px
- Resize persists across sessions.

### 3.2. Auto-size
- Double-click on column edge → auto-fit to content.
- Auto-size uses:
  - header width
  - sample of visible rows
- Auto-size persists.

### 3.3. Reordering
- Columns can be reordered via drag & drop.
- Order persists across sessions.

### 3.4. Visibility
- User can hide/show columns via a column menu.
- Hidden columns remain available for export.
- Visibility persists.

--------------------------------------------------
4. Row Behavior
--------------------------------------------------

### 4.1. Editing
Only the IP column is editable.

Editing rules:
- Inline edit on double-click
- Validation:
  - Must be IPv4 or IPv6
  - Must be public
- On validation success:
  - Row is marked “pending”
  - User can trigger re-analysis
- On failure:
  - Inline error message

### 4.2. Adding Rows
At the bottom of the table:
- A special row: “+ Add IP…”
- Clicking creates a new editable row
- Pasting multiple IPs creates multiple rows

### 4.3. Deleting Rows
Each row has a delete icon:
- Removes the row from the table
- Does NOT delete backend data (frontend-only)

--------------------------------------------------
5. Sorting & Filtering
--------------------------------------------------

### 5.1. Sorting
- Click on column header → sort asc/desc
- Multi-column sorting NOT required
- Sorting persists

### 5.2. Filtering (optional, future)
- Simple text filter per column
- Not required for v1.0

--------------------------------------------------
6. Performance Requirements
--------------------------------------------------
- Must support 500–2000 rows without lag
- Virtualized rendering required
- No full re-render on resize
- Debounced updates (50–100ms)

--------------------------------------------------
7. Selection & Keyboard Support
--------------------------------------------------
- Arrow keys navigate rows
- Enter → edit IP cell
- Delete → remove row
- Ctrl+A → select all (future)
- Shift+click → multi-select (future)

--------------------------------------------------
8. Export Integration
--------------------------------------------------
The DataGrid must integrate with the backend export endpoint.

Rules:
- Export uses backend data, not table state
- Hidden columns are still exported
- Sorting does NOT affect export order

--------------------------------------------------
9. Error States
--------------------------------------------------
Row-level errors:
- Invalid IP
- Private/reserved IP
- Lookup failed

Displayed inline with:
- Red border
- Tooltip message

--------------------------------------------------
10. Future Enhancements
--------------------------------------------------
- Column presets (Minimal / Full / Custom)
- Saved table layouts
- Drag & drop CSV import
- Inline WHOIS preview
