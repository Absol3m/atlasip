# [0.6.0] – 2026-04-19

## Added

- **Cross-platform service mode** – AtlasIP can now run as a headless background service on Windows, macOS, and Linux without launching the GUI.
- **Stable local HTTP API** – The embedded HTTP server is now stable and available when running in service mode or alongside the GUI.
- **Unified i18n system** – All application text (UI, CLI, service messages, errors) is now managed through a single i18n layer backed by structured JSON locale files (`i18n/en-US/`, `i18n/fr-FR/`).
- **French translation (fr-FR)** – Full French translation covering all UI pages, settings, error messages, and system notifications.
- **Dynamic version sync** – The application version displayed in the UI is now read at runtime from Tauri via `getVersion()`, eliminating hardcoded version strings.
- **System tray integration** – Closing the main window now hides the application to the system tray. The tray icon provides Show and Quit actions. Left-click toggles window visibility.
- **OS autostart toggle** – A new Behavior section in Settings allows the user to enable or disable automatic launch at login via the OS autostart mechanism.
- **About modal** – Clicking the version badge in the sidebar footer opens an About modal with application description, feature list, GitHub link, credits, license, and legal notice.
- **GitHub Actions release workflow** – Automated CI/CD pipeline for building and publishing release artifacts across all supported platforms.

## Changed

- **Sidebar collapse behavior** – The collapse button now slides in from the left on hover (Spotify-style), shifting the logo and app name to the right. Collapsed state shows only the expand icon.
- **History page** – All strings are now served through the i18n system.
- **Settings page** – All strings including section titles, field labels, hints, validation errors, and option labels are now served through the i18n system.
- **HybridDurationInput** – Natural language phrases and validation error messages are now fully translated via i18n.
- **i18n architecture** – Locale files migrated from scattered per-module files to a unified `i18n/` directory at the project root, shared between the Rust backend and the Tauri frontend.

## Fixed

- Fixed `LOG_MIN` / `LOG_MAX` in `HybridDurationInput` capturing initial prop values instead of reacting to prop changes (converted to `$derived()`).
- Fixed default locale returning `"en"` instead of `"en-US"` in the Tauri config backend.
- Fixed Windows service compilation errors related to tracing dependencies.

## Removed

- Removed the **Advanced** placeholder section from the Settings page.
- Removed the **About** navigation entry from the sidebar (replaced by the About modal).
- Removed legacy per-module locale files (`locales/en.json`, `locales/fr.json`, `ui/src/lib/i18n/`, `ui/src/lib/locales/fr.ts`).

## Notes

- The local API and service mode are functional but not yet exposed in the GUI as a separate management interface. The application follows a single-process model: launching AtlasIP starts both the GUI and the backend service.
- API key management for third-party OSINT sources (AbuseIPDB, VirusTotal) is planned for 0.7.0.
