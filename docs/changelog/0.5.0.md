# AtlasIP v0.5.0 — Headless Service & Local API  
**Release date:** 2026-XX-XX  
**Type:** Feature Release (Major)

---

## ✨ Nouveautés majeures

### 🔧 Mode Headless (exécution sans interface)
AtlasIP peut désormais fonctionner **sans UI**, en mode service, avec un backend autonome exposant une API locale.  
Ce mode permet l’intégration avec des outils externes, dont une future extension Chrome.

- Nouveau paramètre `headless = true` dans `config.toml`
- Démarrage sans fenêtre Tauri
- Backend Rust autonome
- API HTTP locale toujours active (UI ou headless)
- Support complet Windows / macOS / Linux

---

## 🖥️ Services système (Windows / macOS / Linux)

### Windows (SCM)
- Service Windows natif via `windows-service`
- Commandes :
  - `atlasip.exe install-service`
  - `atlasip.exe uninstall-service`
- Démarrage automatique + exécution headless

### macOS (launchd)
- Support LaunchAgent / LaunchDaemon
- Fichier plist généré automatiquement
- Commandes :
  - `atlasip install-service-macos`
  - `atlasip uninstall-service-macos`

### Linux (systemd)
- Service systemd natif
- Commandes :
  - `atlasip install-service-linux`
  - `atlasip uninstall-service-linux`

## 🌐 API Locale (Axum)
Nouvelle API HTTP locale, accessible uniquement via `127.0.0.1`.

### Endpoints :
- `GET /analyze/ip/:ip`
- `GET /analyze/domain/:domain`
- `GET /reverse/:ip`
- `GET /health`

### Caractéristiques :
- JSON propre et stable
- Toujours disponible (UI ou headless)
- Jamais exposée en externe

---

## 🔎 Reverse DNS via DoH (DNS over HTTPS)
Le reverse DNS utilise désormais un mode moderne et résilient compatible VDI / proxification.

### Modes disponibles :
- **System DNS only**
- **DoH only**
- **Automatic (System → fallback DoH)**
- **Disabled**

### Implémentation :
- Endpoint Cloudflare DoH (`/dns-query`)
- Timeout DNS système configurable
- Fallback intelligent automatique
