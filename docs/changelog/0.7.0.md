# [0.7.0] – 2026-04-25

## Added

- **BGP/ASN enrichment** – New `src/bgp.rs` module queries RIPEstat to resolve the originating ASN, AS name, country, announced IPv4/IPv6 prefixes, and BGP peers for any IP address. Results are embedded in `IpRecord.bgp` and exposed via the HTTP API.
- **GeoIP support** – New `src/geoip.rs` module integrates the MaxMind GeoLite2-City database to provide latitude, longitude, and city for each IP. The database is auto-downloaded and cached locally (`~/.config/atlasip/geoip/`).
- **DNSSEC validation** – DNS records now carry a `dnssec_validated` flag (AD bit from the resolving DoH response), indicating whether the record passed DNSSEC chain-of-trust validation.
- **Extended DNS record types** – DNS lookups now query MX, NS, and SOA records in addition to A, AAAA, CNAME, and TXT.
- **DNS-over-TLS (DoT) transport** – Added support for DNS-over-TLS as a selectable DNS mode alongside the existing system and DoH transports.
- **First-launch modal** – New `FirstLaunchModal.svelte` component guides users through initial setup on first run.
- **Reverse-IP endpoint** – New `/reverse-ip/:ip` HTTP route for reverse domain lookups.

## Changed

- **IP detail page overhauled** – `ui/src/routes/ip/[ip]/+page.svelte` significantly expanded to display BGP, GeoIP, and DNSSEC data alongside existing RDAP/WHOIS information.
- **Parallel enrichment pipeline** – RDAP, WHOIS, and BGP lookups now run concurrently via `tokio::join!`, reducing total lookup latency.
- **DNS error handling improved** – A missing A/AAAA record no longer aborts the full lookup; MX/NS/TXT results are preserved and the error is surfaced as a non-fatal warning.
- **RDAP client refactored** – Significant simplification of `src/rdap/client.rs` for maintainability.
- **Settings page** – Updated to expose new DNS mode and GeoIP configuration options.
- **i18n (en-US + fr-FR)** – All new UI strings translated in both locales.
