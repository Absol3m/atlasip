# [0.7.1] – 2026-04-25

Stabilisation patch following 0.7.0. Full backend and frontend audit, silent bug fixes, and a history page UX overhaul.

---

## Backend — Critical fixes

- **GeoIP reader cached** — the `.mmdb` file was reopened on every lookup. It is now loaded once at server startup via `OnceLock` and reused for all subsequent lookups.
- **`geo_country` now stored** — the country field returned by GeoLite2 was extracted but never written to `IpRecord`. Fixed; available in the JSON response as `geo_country`.
- **PTR respects `dns_mode`** — the PTR reverse lookup inside `full_dns_lookup` always used the system resolver, ignoring the configured mode (DoH, DoT, Disabled). Fixed via `reverse_lookup_smart`.
- **DNSSEC silent in DoT mode** — DNSSEC validation was always performed via a DoH request even when `DotOnly` was configured. The check is now skipped in that mode to avoid unsolicited DoH traffic.

## Backend — High-priority fixes

- **Session record store bounded** — `AppState.records` (`Vec<IpRecord>`) grew without limit. Now capped at 10,000 entries with FIFO eviction.
- **Proxy applied to HTTP clients** — `AppConfig.proxy.*` settings were saved but ignored by all HTTP clients (`BgpClient`, `reverse_ip_domains`, hackertarget, crt.sh). An `apply_proxy()` helper is now used across the board.

## Backend — Medium fixes

- **`ConfigUpdateRequest` accepts GeoIP fields** — `maxmind_account_id` and `maxmind_license_key` can now be updated via `POST /config`.
- **BGPView comment corrected** — the code uses RIPEstat; the comment said BGPView.
- **`reverse_ip_domains` user-agent corrected** — truncated GitHub URL replaced with the full repository URL.

## Backend — Cleanup

- **`GET /health` exposes GeoIP database status** — returns `{ "status": "ok", "geoip_db": "ok"|"outdated"|"missing" }`.
- **`apply_rdap` and `apply_whois` unified** — the two identical functions replaced by a single `apply_parsed_fields` with a shared `ParsedFields` struct.
- **`pause_every` / `pause_duration_ms` implemented** — rate-limiting in `lookup_bulk` is now functional.
- Dead code removed: `split_prefixes`, `is_hostname`; `db_status` promoted to the `/health` endpoint.

---

## Frontend — Critical fixes

- **`geo_country` added to the TypeScript `IpRecord` interface** — the field existed on the Rust side but was absent from the TypeScript type.
- **`init_reader()` called after GeoIP download** — a freshly downloaded GeoLite2 database was inactive until the next restart. Fixed.
- **`first_launch` reset after modal dismissed** — the first-launch modal would reappear if the config file was missing or unparseable. Fixed.
- **Settings synced to the backend** — `locale`, `dns_timeout_ms`, `rdap_timeout_ms`, `whois_timeout_ms`, `maxmind_account_id`, and `maxmind_license_key` are now sent to the backend HTTP API (`POST /config`) on every settings save.

## Frontend — High-priority fixes

- **`BASE_URL` centralised** — the constant `http://127.0.0.1:8080` was duplicated in `lookup.ts` and `atlasip.ts`. Now exported from `atlasip.ts` and imported in `lookup.ts`.
- **`getHealth()` returns `geoip_db`** — the `geoip_db` field added to the `/health` endpoint was silently dropped by the TypeScript service. Now exposed via `HealthStatus`.
- **Column labels translated** — table headers (`IP Address`, `Country`, `Owner`…) are now resolved via `i18n.t('table.col.*')` and follow the active locale.

## Frontend — Medium fixes

- **Default visible columns reduced** — the table opened with all 20 columns visible. Reduced to 10 essential columns; the rest remain accessible via the column drawer.
- **localStorage history validation** — corrupted or legacy entries are now filtered out on load.
- **`scheduleparse` → `scheduleParse`** — renamed to follow camelCase convention.

## Frontend — Refactoring

- **`$i18n` alias** — locale JSON imports now use the `$i18n/en-US/...` alias instead of fragile four-level relative paths.
- **`BgpCard.svelte` and `DnsRecordsCard.svelte` extracted** — `ip/[ip]/+page.svelte` reduced from 1,557 to 1,190 lines; CSS encapsulated in each component.
- **Stale `@ts-expect-error` removed** from `vite.config.js` — TypeScript now recognises `process` without the directive. The check passes with 0 errors / 0 warnings.

---

## History page — UX overhaul

- **Grouped by target** — one card per unique IP or domain instead of one card per consultation. Duplicates are gone.
- **N× badge** — consultation count displayed as a badge on the grouped card.
- **Disclosure triangle (▶/▼)** — chevron on the left below the target chips, with a `N consultations — last: date` line. Expanding reveals previous consultations with their absolute date and result count.
- **Card click to rerun** — clicking the card reruns the analysis, as before. The chevron intercepts the click without propagating it.
- **Individual delete buttons removed** — history is local-only; individual deletion adds no real value. Only the global Clear button is kept.
- **Header counter** — now shows the number of unique targets rather than the raw entry count.
