# [0.7.1] – 2026-04-25

Patch de stabilisation post-0.7.0. Audit complet du backend et du frontend, corrections de bugs silencieux, améliorations UX sur la page historique.

---

## Backend — Corrections critiques

- **GeoIP reader mis en cache** — le fichier `.mmdb` était rouvert à chaque lookup. Il est désormais chargé une seule fois au démarrage du serveur via `OnceLock` et réutilisé pour tous les lookups.
- **`geo_country` maintenant stocké** — le champ pays retourné par GeoLite2 était extrait mais jamais écrit dans `IpRecord`. Corrigé ; disponible dans la réponse JSON sous `geo_country`.
- **PTR respecte `dns_mode`** — le reverse lookup PTR à l'intérieur de `full_dns_lookup` utilisait toujours le resolver système, ignorant le mode configuré (DoH, DoT, Disabled). Corrigé via `reverse_lookup_smart`.
- **DNSSEC silencieux en mode DoT** — la validation DNSSEC passait toujours par une requête DoH même quand l'utilisateur avait choisi `DotOnly`. La vérification est maintenant désactivée dans ce mode pour ne pas envoyer de requêtes DoH non sollicitées.

## Backend — Corrections hautes

- **Taille des résultats en session bornée** — `AppState.records` (`Vec<IpRecord>`) grossissait indéfiniment. Désormais plafonné à 10 000 entrées avec éviction FIFO.
- **Proxy appliqué aux clients HTTP** — les paramètres `proxy.*` de `AppConfig` étaient sauvegardés mais ignorés par tous les clients HTTP (`BgpClient`, `reverse_ip_domains`, hackertarget, crt.sh). Un helper `apply_proxy()` est désormais utilisé partout.

## Backend — Corrections moyennes

- **`ConfigUpdateRequest` accepte les champs GeoIP** — `maxmind_account_id` et `maxmind_license_key` sont maintenant transmissibles via `POST /config`.
- **Commentaire BGPView corrigé** — le code utilise RIPEstat, le commentaire disait BGPView.
- **User-agent `reverse_ip_domains` corrigé** — URL GitHub tronquée remplacée par l'URL complète.

## Backend — Nettoyage

- **`GET /health` expose l'état de la base GeoIP** — retourne `{ "status": "ok", "geoip_db": "ok"|"outdated"|"missing" }`.
- **`apply_rdap` et `apply_whois` unifiés** — les deux fonctions identiques remplacées par `apply_parsed_fields` + `ParsedFields` commun.
- **`pause_every` / `pause_duration_ms` implémentés** — le rate-limiting dans `lookup_bulk` est désormais fonctionnel.
- Suppression du dead code : `split_prefixes`, `is_hostname`, `db_status` (exposée via `/health`).

## Frontend — Corrections critiques

- **`geo_country` ajouté à l'interface TypeScript `IpRecord`** — le champ existait côté Rust mais était absent du type TypeScript.
- **`init_reader()` appelé après téléchargement GeoIP** — la base GeoLite2 téléchargée via Settings était inactive jusqu'au redémarrage. Corrigé.
- **`first_launch` remis à `false` après fermeture de la modale** — la modale de premier lancement réapparaissait si la config était absente ou corrompue.
- **Settings synchronisés vers le backend** — `locale`, `dns_timeout_ms`, `rdap_timeout_ms`, `whois_timeout_ms`, `maxmind_account_id` et `maxmind_license_key` sont maintenant envoyés au backend HTTP (`POST /config`) à chaque sauvegarde des paramètres.

## Frontend — Corrections hautes

- **`BASE_URL` centralisé** — la constante `http://127.0.0.1:8080` était dupliquée dans `lookup.ts` et `atlasip.ts`. Elle est maintenant exportée depuis `atlasip.ts` et importée dans `lookup.ts`.
- **`getHealth()` retourne `geoip_db`** — le champ `geoip_db` ajouté au endpoint `/health` était ignoré par le service TypeScript.
- **Labels de colonnes traduits** — les en-têtes du tableau (`IP Address`, `Country`, `Owner`…) sont désormais résolus via `i18n.t('table.col.*')` et suivent la locale active.

## Frontend — Corrections moyennes

- **Colonnes visibles par défaut réduites** — la table s'ouvrait avec les 20 colonnes visibles. Réduit à 10 colonnes essentielles ; le reste accessible via le tiroir de colonnes.
- **Validation du localStorage pour l'historique** — les entrées corrompues ou d'une ancienne version sont filtrées au chargement.
- **`scheduleparse` → `scheduleParse`** — renommage pour respecter la convention camelCase.

## Frontend — Refactoring

- **Alias `$i18n`** — les imports de fichiers JSON de traduction utilisent maintenant l'alias `$i18n/en-US/...` au lieu de chemins relatifs traversant 4 niveaux.
- **`BgpCard.svelte` et `DnsRecordsCard.svelte` extraits** — la page `ip/[ip]/+page.svelte` passe de 1557 à 1190 lignes ; les styles CSS sont encapsulés dans chaque composant.
- **`@ts-expect-error` obsolète retiré** de `vite.config.js` — TypeScript reconnaît maintenant `process` sans directive. Le check passe à 0 erreur / 0 warning.

## Historique — Refonte UX

- **Groupement par cible** — une card unique par IP ou domaine au lieu d'une card par consultation. Les doublons disparaissent.
- **Badge N×** — le nombre de consultations est affiché en badge sur la card groupée.
- **Disclosure triangle (▶/▼)** — chevron à gauche sous les chips cibles, avec la ligne `N consultations — dernière : date`. Le déroulé affiche les consultations précédentes avec leur date absolue et leur nombre de résultats.
- **Card cliquable pour relancer** — clic sur la card = relance l'analyse, comme avant. Le chevron intercepte le clic sans le propager.
- **Suppression des boutons X individuels** — l'historique est local, la suppression individuelle n'apporte pas de valeur. Seul le bouton « Clear » global est conservé.
- **Compteur d'en-tête** — affiche le nombre de cibles uniques plutôt que le nombre brut d'entrées.
