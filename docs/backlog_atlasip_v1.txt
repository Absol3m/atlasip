1. INTRODUCTION
-------------------------------------------
AtlasIP est un outil OSINT moderne permettant d’analyser des IPs, hostnames, URLs, email headers et données réseau. 
Ce document définit le backlog complet du projet, incluant les spécifications, les bugs connus, les règles de parsing, les tâches de développement, les priorités et la roadmap.

Ce fichier doit être considéré comme la source de vérité. Toute modification doit être versionnée.


2. OBJECTIFS GLOBAUX D’ATLASIP
-------------------------------------------
- Offrir un outil OSINT moderne, rapide, fiable et agréable à utiliser.
- Supporter IPv4, IPv6, hostnames, URLs, email headers et texte libre.
- Fournir un parsing strict, cohérent et testable.
- Offrir une UX professionnelle, moderne, intuitive.
- Fournir des résultats RDAP/WHOIS/DNS fiables et enrichis.
- Permettre l’export, la manipulation et l’analyse avancée des données.


3. CHAPITRE 1 — PARSING & VALIDATION (v2.0)
-------------------------------------------
3.1 Types d’entrées supportées
- IPv4 strict
- IPv6 strict
- Hostnames stricts
- URLs (avec protocole, port, path, query)
- Texte libre contenant IPs, hostnames, URLs
- Email headers (extraction d’IP)
- Entrées multi-lignes

3.2 Types d’entrées à rejeter
- IPv4 invalides (trop d’octets, hors plage)
- IPv6 invalides
- Hostnames invalides
- URLs invalides
- Mots normaux (and, hello, test)
- Entrées tronquées
- Entrées vides

3.3 Règles de parsing strictes
- IPv4 : regex stricte + validation numérique
- IPv6 : parser standard
- Hostname : regex stricte
- URL : parser standard + extraction du hostname
- Texte libre : extraction IP + hostname + URL + header
- Ignorer les mots normaux et la ponctuation

3.4 Validation globale
- Une seule entrée invalide bloque tout
- Bouton Analyze = gris
- Badge = rouge
- Message d’erreur global obligatoire
- Aucune analyse possible tant que tout n’est pas valide

3.5 Détection du mode
- IPv4 only → IPv4 DETECTED
- IPv6 only → IPv6 DETECTED
- Hostname only → HOSTNAME DETECTED
- Mix IP + hostname → MIXED MODE
- Mix IP + URL → MIXED MODE
- Mix hostname + URL → MIXED MODE
- Mix IPv4 + IPv6 → MIXED MODE
- Erreur → INVALID

3.6 Compteur cohérent
- Compter IPs, hostnames, URLs
- Ne jamais ignorer une entrée valide
- Ne jamais ignorer une entrée invalide

3.7 Bugs identifiés (à corriger)
- Mixed Mode non implémenté
- Hostnames ignorés en présence d’IPs
- URLs non parsées
- IP invalides non bloquantes
- Mots normaux traités comme IP invalides
- Badge incorrect
- Message d’erreur global manquant
- Compteur incorrect
- Parsing non strict
- Tronquage silencieux
- Extraction hostname depuis URL manquante
- Feedback utilisateur incohérent

3.8 Tâches du Sprint 2.2.1 — Parsing v2
- Implémenter la spec complète
- Implémenter la validation globale
- Implémenter la détection du mode
- Implémenter le compteur cohérent
- Implémenter les messages d’erreur
- Implémenter les tests unitaires (50+)
- Implémenter les cas limites
- Implémenter les cas mixtes
