AtlasIP — Outil d’OSINT IP moderne, inspiré d’IPNetInfo (NirSoft)

Version : 0.2
Auteur : Kévin
Objectif : Moderniser IPNetInfo en conservant ses fonctionnalités clés, tout en ajoutant un backend moderne (DNS + RDAP + WHOIS), une API locale, des exports avancés et une intégration future avec une UI (Tauri) et une extension navigateur.

--------------------------------------------------
1. Objectifs fonctionnels
--------------------------------------------------

1.1. But principal

AtlasIP est un outil d’OSINT focalisé sur les adresses IP publiques, destiné à :
- Identifier le propriétaire d’une IP (organisation, netname, range).
- Récupérer les informations de contact (adresse, téléphone, fax, email, abuse).
- Identifier le pays / région / registre.
- Aider à l’analyse d’emails (origine d’un spam, d’un phishing, etc.).
- Fournir une API locale pour une UI moderne et une extension navigateur.

1.2. Héritage IPNetInfo (NirSoft)

AtlasIP doit reprendre les comportements clés d’IPNetInfo :
- Ignorer les IP non publiques (privées, loopback, link-local, multicast, documentation, etc.).
- Extraire des IP à partir de headers d’emails (mode futur / optionnel côté front).
- Interroger d’abord ARIN, puis les autres RIR (RIPE, APNIC, LACNIC, AFRINIC) si nécessaire.
- Analyser le WHOIS brut et présenter les données dans un tableau structuré.
- Exporter les résultats (CSV, tab-delimited, etc.).
- Supporter les proxys (SOCKS4/5) pour les requêtes réseau.
- Gérer les erreurs réseau et les limitations de rate-limit (WHOIS).

1.3. Modernisation par rapport à IPNetInfo

AtlasIP ajoute :
- Un backend Rust moderne (Tokio, Axum, reqwest, hickory-resolver).
- Un pipeline DNS + RDAP + WHOIS (RDAP prioritaire, WHOIS en fallback).
- Une API HTTP locale pour intégration UI / extension navigateur.
- Des exports multiples (CSV, TSV, TXT, HTML, XML, WHOIS brut).
- Une configuration persistante (config.toml).
- Une internationalisation (locales JSON).
- Des tests unitaires et d’intégration.

--------------------------------------------------
2. Comportement réseau et pipeline
--------------------------------------------------

2.1. Filtrage des IP non publiques (héritage IPNetInfo)

AtlasIP ne doit traiter que des IP publiques.

Les IP suivantes doivent être ignorées / rejetées :

- Privées (RFC 1918) :
  - 10.0.0.0 – 10.255.255.255
  - 172.16.0.0 – 172.31.255.255
  - 192.168.0.0 – 192.168.255.255

- Loopback :
  - 127.0.0.0 – 127.255.255.255

- Link-local (APIPA) :
  - 169.254.0.0 – 169.254.255.255

- Multicast :
  - 224.0.0.0 – 239.255.255.255

- Documentation / réservées :
  - 192.0.2.0/24
  - 198.51.100.0/24
  - 203.0.113.0/24
  - 240.0.0.0/4
  - 255.255.255.255

Comportement :
- En mode API (lookup IP / hostname) : retourner une erreur 422 “Private or reserved IP not allowed”.
- En mode bulk : ces IP sont silencieusement filtrées par `clean_targets()` (elles ne sont pas envoyées au pipeline).
- En interne : le pipeline DNS/RDAP/WHOIS ne doit jamais être appelé pour ces IP.

2.2. Pipeline de résolution (public IP uniquement)

Pour une IP publique ou un hostname résolu vers une IP publique :

Étapes :

1. DNS forward (si hostname)
   - `dns::resolve_hostname()` pour obtenir une IP (A/AAAA).
   - Si la résolution donne une IP non publique → 422 (hostname résout vers IP privée).

2. PTR lookup (reverse DNS)
   - `dns::reverse_lookup()` pour obtenir un nom résolu.
   - Échec non fatal (NXDOMAIN → simplement pas de nom).

3. RDAP (prioritaire)
   - Utiliser `rdap::client::query()` pour récupérer les données structurées.
   - Remplir les champs : owner, country, range, status, abuse, etc.
   - `whois_source = "RDAP"`.

4. WHOIS fallback
   - Si RDAP échoue ou est indisponible, utiliser `whois::client::query()`.
   - Interroger d’abord ARIN, puis suivre les referrals vers RIPE/APNIC/LACNIC/AFRINIC.
   - Parser les formats ARIN-style et RIPE-style.
   - `whois_source = <serveur WHOIS>`.

2.3. Proxys et timeouts

- Toutes les requêtes HTTP (RDAP, bootstrap IANA) passent par `reqwest::Client` avec :
  - Timeout global configurable.
  - Support optionnel des proxys SOCKS4/SOCKS5 (sans auth dans un premier temps).

- Toutes les requêtes WHOIS passent par `tokio::net::TcpStream` avec :
  - Timeout via `tokio::time::timeout`.
  - Gestion des erreurs réseau propre (pas de panic, pas de unwrap).

2.4. Gestion des erreurs et rate-limit

- Si un serveur WHOIS renvoie “Query rate limit exceeded” :
  - Marquer la requête comme échouée proprement.
  - Ne pas réessayer en boucle.
- Si ARIN est indisponible :
  - Tenter les autres RIR si possible.
  - Sinon, retourner une erreur claire.

--------------------------------------------------
3. Modèle de données
--------------------------------------------------

3.1. IpRecord

Structure centrale représentant une ligne de résultat.

Champs (inspirés d’IPNetInfo + enrichis) :

- id : UUID
- ip : String
- hostname : Option<String>
- resolved_name : Option<String>
- is_private : bool (true si IP non publique, mais normalement filtrée en amont)
- country : Option<String>
- state : Option<String>
- city : Option<String>
- owner_name : Option<String>
- org_name : Option<String>
- netname : Option<String>
- cidr : Option<String>
- range_start : Option<String>
- range_end : Option<String>
- abuse_emails : Vec<String>
- contact_emails : Vec<String>
- phone : Option<String>
- fax : Option<String>
- address : Option<String>
- created_at : Option<String>
- updated_at : Option<String>
- whois_source : Option<String>   // "RDAP", "whois.arin.net", etc.
- raw_whois : Option<String>

3.2. COLUMNS et to_row()

- `COLUMNS: &[&str]` — liste ordonnée des 21 colonnes (héritage IPNetInfo + spec AtlasIP).
- `IpRecord::to_row()` — retourne `Vec<String>` dans l’ordre de `COLUMNS`.
  - Les champs multi-valeurs (emails) sont joints avec `"; "`.
  - Les champs optionnels vides → chaîne vide.

--------------------------------------------------
4. API HTTP locale (Axum)
--------------------------------------------------

4.1. AppState

- `config: Arc<RwLock<AppConfig>>`
- `records: Arc<RwLock<Vec<IpRecord>>>`

4.2. Endpoints

- `GET /health`
  - Retourne `{ "status": "ok" }`.

- `GET /lookup/ip/:ip`
  - Valide l’IP.
  - Si IP non publique → 422.
  - Lance le pipeline (DNS PTR + RDAP + WHOIS fallback).
  - Stocke le résultat dans `records`.
  - Retourne `IpRecord` en JSON.

- `GET /lookup/hostname/:host`
  - Résout le hostname en IP.
  - Si IP non publique → 422.
  - Pipeline identique à `/lookup/ip`.

- `POST /lookup/bulk`
  - Corps : liste de cibles (IP/hostnames).
  - `clean_targets()` :
    - normalise
    - déduplique
    - filtre les IP non publiques
  - Exécute les lookups en parallèle (JoinSet).
  - Stocke les résultats dans `records`.
  - Retourne la liste des `IpRecord`.

- `GET /export`
  - Query params : `format`, `ids` (optionnel).
  - Filtre `records` par IDs si fournis.
  - Délègue à `export::export(records, format)`.
  - Retourne le bon `Content-Type`.

- `GET /config`
  - Retourne la config courante en JSON.

- `POST /config`
  - Merge partiel des champs.
  - TODO : persistance sur disque.

4.3. CORS

- CORS permissif pour permettre à une extension navigateur (Chrome/Firefox) d’appeler l’API locale.

--------------------------------------------------
5. Exports
--------------------------------------------------

5.1. Formats supportés

- CSV (séparateur `,`, header optionnel).
- TSV (tab-delimited).
- TXT (vertical, type “fiche”).
- TXT (horizontal, type “table”).
- HTML (table).
- HTML (fiche détaillée).
- XML.
- WHOIS brut (concaténation des `raw_whois`).

Tous les formats s’appuient sur :
- `COLUMNS`
- `IpRecord::to_row()`

5.2. Options héritées d’IPNetInfo

- Ajout d’une ligne d’en-tête (activé par défaut).
- Export CSV/Tab depuis la sélection (côté front futur).
- Possibilité d’ouvrir l’adresse dans Google Maps (côté UI, pas backend).

--------------------------------------------------
6. Configuration
--------------------------------------------------

6.1. Fichier config.toml

Champs possibles :

- `listen_addr` : "127.0.0.1:8080"
- `default_timeout_ms` : u64
- `proxy_socks4` : Option<String>
- `proxy_socks5` : Option<String>
- `ignore_private_ips` : bool (par défaut true, pour rester fidèle à IPNetInfo)
- `max_concurrent_lookups` : u32

6.2. Chargement / sauvegarde

- `AppConfig::load(path)` — charge depuis TOML.
- `AppConfig::save(path)` — sauvegarde.

--------------------------------------------------
7. Internationalisation
--------------------------------------------------

- `i18n::t(key: &str) -> String`.
- Utilisé pour les messages d’erreur, labels, etc.

--------------------------------------------------
8. Extension navigateur (futur)
--------------------------------------------------

- Extension Chrome/Firefox qui :
  - Récupère l’IP du site courant.
  - Appelle `GET /lookup/ip/:ip` sur `127.0.0.1:8080`.
  - Affiche les infos (owner, country, abuse, etc.).
- Nécessite que AtlasIP tourne en tâche de fond (mode service possible).
